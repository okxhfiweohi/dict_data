# 规范

## 规范格式说明

同一个词的多个重定向分开;
写

```plain
a => a_1
a => a_2
```

单词不能有特殊符号

```plain
/* 不能是 colo(u)r := ... */
color := ... /* 分开写 */
colour => color /* 重定向到color */
```

同一个词的解释写成 1 行, 用 `||` 分隔

每个解释构成格式

```plain
[<词性> ]["<"<性质>">" ]["["<门类>"]"] 解释内容
```

词性是以下几种: (后缀点)

|词性缩写 | 英文全称 | 中文释义 |
|----|----|----|
| n. | Noun | 名词 |
| a. | Adjective | 形容词 |
| v. | Verb | 动词 |
| adv. | Adverb | 副词 |
| abbr. | Abbreviation | 缩写词 |
| int. | Interjection | 感叹词 |
| prep. | Preposition | 介词 |
| num. | Numeral | 数词 |
| pron. | Pronoun | 代词 |
| conj. | Conjunction | 连词 |
| phr. | Phrase | 短语 |
| det. | Determiner | 限定词（含冠词） |

通常每个解释都要有词性, 只有在不存在于上述列表中可不标注词性,多个词性可以合写:

```plain
swim := v.,n. 游泳
```

性质是下面几种:

- U : 不可数
- C : 可数
- P : 复数
- I : 不及物
- T : 及物
- M : 情态动词

门类必须使用上述列表中的缩写形式, 其他的不规范, 需要改写规范(或者考虑改写成`()`补充说明).
门类可以不写,也可以多个之间紧密排列 :

```plain
[X][Y][Z] explain
```

解释内容是单词或短语的中文含义
不能是或者包含:

- 音标
- 例句
- `[]` 英文中括号, 会干扰前面的门类
- `<>` 英文尖括号, 会干扰前面的性质
- 无意义的符号

可以有:

- `()` 英文中括号, 补充说明信息
- `,` `;` `""` 等常用符号

词条解释建议:

- 使用 `,` 分隔相同含义,不同的中文解释
- 使用 `;` 分隔出不同但相近的含义(意义的延伸)
- 如果意思差别非常大, 或者词性和意思不同, 使用 `||` 拆分成两个条目
